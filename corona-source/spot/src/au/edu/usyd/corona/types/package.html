Contains both the interfaces and actual implementations of {@link au.edu.usyd.corona.srdb} data types that are used throughout the system.

Package Specification

SRDB, the SunSPOT Relational Database layer that we have implemented, has its own data type definition. Each of these data types have certain operators defined, and for all intensive purposes are valid data types as in any programming language.

The default data types we provide (and use) within the system are:

  • {@link au.edu.usyd.corona.types.BooleanType}
  • {@link au.edu.usyd.corona.types.ByteType}
  • {@link au.edu.usyd.corona.types.FloatType}
  • {@link au.edu.usyd.corona.types.IEEEAddressType}
  • {@link au.edu.usyd.corona.types.IntType}
  • {@link au.edu.usyd.corona.types.LongType}

If someone wishes to add a new data type to the system, all they have to do is create a class that implements {@link au.edu.usyd.corona.types.ValueType}. This interface defines the operators and the the other required methods that a SRDB data type must have. An SRDB data type must also have a default constructor.

When implementing operators, if data type X can not perform operation A on data type Y, then in the implementation of operator A in class X, if the argument to operator A is of type Y, then a {@link au.edu.usyd.corona.types.InvalidOperationException} should be thrown.
